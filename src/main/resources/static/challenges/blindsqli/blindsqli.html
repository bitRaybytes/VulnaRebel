Blind SQL Injection

Almost silent - but not quite.

Difficulty: Medium


Objective:
This user search form is connected to a database. It only ever responds with "User found" or "User not found" - nothing more.

Somewhere in the database is a secret column that is never shown to you directly. Your goal is to extract its value one character at a time and submit the flag below.

{{resultDisplay}}
Need a hint? reset hints

The application only tells you true or false. What if you could turn a SQL query into a yes/no question?
More hints

The query behind this form looks something like:
SELECT username FROM blind_users WHERE username = 'your_input'
It returns nothing visible - just whether a row was found. Can you append a condition that tests a single character?
One more hint

Look up SUBSTRING(column, position, 1) in PostgreSQL.
If you can make the query return "User found" only when a specific character matches, you can extract the secret one character at a time.
Example shape (fill in the blanks):
admin' AND SUBSTRING(secret, 1, 1)='?' --
